## 2. 项目背景与目标

### 2.1 背景
原系统为“中药制剂中非法添加高风险物质快速筛查系统 V1.0”，基于 HPLC-DAD（高效液相色谱-二极管阵列检测器）方法，用数字对照品库替代实体对照品进行现场快筛。原系统为桌面端，技术栈为 Python + tkinter + 阿里云 MySQL，已实现：

- 用户登录（界面有管理员/普通用户选项，但登录校验函数为空）。
- 数据导入（txt / xlsx）。
- 三种检测模型：相对保留时间、峰面积比值、紫外吸收光谱。
- 数据库查询、更新、重置、导出。
- 日志生成。

### 2.2 目标
将系统改造为 B/S 架构网页端：

1. 前端采用 Vue 3（可配合 Element Plus / ECharts）。
2. 后端采用 Flask（RESTful API）。
3. 数据库采用 MySQL（本地或云端均可配置）。
4. 保留并完善原有检测模型与数据库管理功能。
5. 新增 **多药物批量检测** 与 **峰值图对比** 功能。
6. 实现完整的用户鉴权、权限控制、操作日志与数据备份。

## 3. 术语与约定

| 术语 | 说明 |
|------|------|
| HPLC-DAD | 高效液相色谱-二极管阵列检测器。 |
| 保留时间（RT） | 色谱峰出现的时间，单位：分钟。 |
| 参照物 | 某类药物中用于计算相对保留时间的基准物质。 |
| 相对保留时间（RRT） | `样品峰保留时间 / 参照物保留时间`。 |
| 峰面积 | 某波长下色谱峰的面积响应值。 |
| 峰面积比值 | 多波长峰面积之间的比值，例如 `A245/A250`、`A255/A250`、`A260/A250`。 |
| 紫外吸收光谱（UV） | 色谱峰的最大吸收波长（Lambda max），可能有 1~3 个值，用 `/` 分隔。 |
| 数字对照品库 | 将上述三类数据按药物类别整理后的结构化数据库。 |
| 检测任务 | 一次完整的检测过程，可包含一个或多个样品、一个或多个峰。 |
| 峰值图 | 以保留时间为横轴、响应值（面积/吸光度）为纵轴的色谱峰可视化图形。 |

## 5. 功能需求

### 5.1 登录与鉴权

- FR-AUTH-01：用户通过用户名、密码登录；密码需加密存储（bcrypt / PBKDF2）。
- FR-AUTH-02：后端返回 Token（JWT），前端在后续请求中携带 Token。
- FR-AUTH-03：管理员可新增/编辑/禁用/删除用户；用户角色只能是 `admin` 或 `operator`。
- FR-AUTH-04：连续失败 5 次后锁定账户 15 分钟或需要验证码。

### 5.2 数据导入

#### 5.2.1 导入方式
- FR-IMPORT-01：支持 txt 与 xlsx 两种格式批量导入。
- FR-IMPORT-02：支持“单文件导入”与“多文件批量导入”。
- FR-IMPORT-03：支持手动录入一个样品的数据（保留时间、峰面积、UV）。
- FR-IMPORT-04：导入时校验文件格式，格式错误给出具体行号与错误信息。

#### 5.2.2 文件格式要求

**A. 保留时间 txt（每组实验一个文件，文件名即为试验编号）**

```text
'保留时间','分钟'
'('#')','#'
'1','6.481'
'2','7.021'
...
```

- 文件名为试验编号（如 `0406-1`）。
- `# #` 行为分隔符，其后为按顺序排列的峰保留时间。
- 峰序号从 1 开始，保留时间单位为分钟。

**B. 峰面积 txt（每个文件包含 4 组试验数据，按波长递增顺序排列）**

文件头示例：

```text
'进样','样品名称','减肥','种混梯','样品瓶','采集日期','面积',...
'2998 PDA 245.0 纳米 (2998 (190-400) 纳米)',...
...
'#','#'
'1','413453'
...
': 2998 ','PDA 248.0 ',' ...
'#','#'
'1','252677'
...
```

- 一个 txt 内依次包含 4 个波长的峰面积数据块。
- 每个数据块以 `: 2998` + `PDA <波长>.0` 开头，随后 `# #` 分隔，再列出 `峰号,面积`。
- 系统按类别配置读取对应波长（见第 6 章）。

**C. 紫外吸收光谱 xlsx**

- 从第二列起，每一列为一组试验；第一行为试验编号。
- 每一行对应一个峰，单元格内为最大吸收波长；若一个峰有多个最大吸收值，用 `/` 分隔。
- 示例：

| 峰的最大吸收值 | 样品1 | 样品2 | ... |
|----|----|----|----|
| 峰1 | 230.99 | 232.17 | ... |
| 峰2 | 290.26 | 290.26 | ... |
| 峰3 | 227.45/266.49 | 226.3/266.49 | ... |

### 5.3 检测模型（核心）

系统必须支持三种检测模型，且按以下顺序组合使用：

1. **相对保留时间初筛（rrt）**
2. **紫外吸收光谱确认（uv_spectrum）**
3. **峰面积比值确认（peak_area）**

模型枚举名与数据库 `detection_results.detection_method` 保持一致。

#### 5.3.1 相对保留时间检测

- FR-DET-01：用户选择药物类别并输入参照物保留时间；或从库中读取该类别的默认参照物。
- FR-DET-02：对每个样品峰计算 `RRT = 峰保留时间 / 参照物保留时间`。
- FR-DET-03：将计算值与该类别的数字对照品库中各物质的 RRT 期望值比较，落在阈值范围内（默认 ±0.001，可配置）则列为可疑物质。
- FR-DET-04：输出每个峰匹配到的物质列表及匹配概率/置信度。

#### 5.3.2 紫外吸收光谱检测

- FR-DET-05：输入峰的最大吸收波长（1~3 个）。
- FR-DET-06：与库中该类别各物质的 UV Lambda max 计算欧氏距离。
- FR-DET-07：距离小于阈值则判定为匹配；超过阈值概率为 0。
- FR-DET-08：输出匹配物质及距离值。

#### 5.3.3 峰面积比值检测

- FR-DET-09：输入一个峰在多个波长下的峰面积（如 A245、A250、A255、A260，具体波长按类别配置）。
- FR-DET-10：计算峰面积比值（如 `R1=A245/A250`、`R2=A255/A250`、`R3=A260/A250`）。
- FR-DET-11：将比值向量与库中各物质的期望向量及协方差矩阵代入贝叶斯/正态模型，计算匹配概率。
- FR-DET-12：输出匹配物质及概率。

#### 5.3.4 综合判定

- FR-DET-13：对同一个峰，综合三种模型的结果生成最终结论：
  - 若三种模型均指向同一物质 → **确认**。
  - 若部分模型匹配 → **可疑**。
  - 若无模型匹配 → **未检出**。
- FR-DET-14：每个结论必须附带置信度或匹配依据。

### 5.4 多药物批量检测（新增）

#### 5.4.1 输入

- 支持一次上传多个样品文件（保留时间 txt、峰面积 txt、UV xlsx 可分别批量上传，或按样品打包）。
- 支持为整批样品统一选择药物类别，也支持每个样品单独选择类别。
- 支持输入/读取参照物保留时间（可按类别默认）。

#### 5.4.2 处理流程

1. 后端解析每个文件，生成样品记录与峰记录。
2. 对每个样品的每个峰执行三种模型检测。
3. 汇总每个样品的疑似非法添加物质列表。

#### 5.4.3 输出

- 批量检测结果表格：
  - 样品编号、药物类别、峰序号、保留时间、RRT、UV 匹配、峰面积匹配、综合结论、置信度。
- 每个样品的详细报告页。
- 支持导出 Excel / PDF 报告。
- 若检出非法物质，系统高亮显示并弹出/标记告警。

### 5.5 峰值图对比（新增）

#### 5.5.1 输入

- 样品峰数据：峰序号、保留时间、峰面积（至少一个波长）或 UV 数据。
- 可选对照品：从数字对照品库中选择 1~N 个物质。

#### 5.5.2 功能

- FR-GRAPH-01：在同一坐标系中绘制样品峰与选中对照品的峰位置/形状对比图。
- FR-GRAPH-02：横轴为保留时间，纵轴为响应值（面积或相对强度）。
- FR-GRAPH-03：支持按波长切换显示。
- FR-GRAPH-04：支持缩放、平移、导出图片（PNG）。
- FR-GRAPH-05：在图上标注匹配峰与物质名称。

#### 5.5.3 输出

- 可视化对比图。
- 图例说明：样品峰、各对照品峰、匹配/未匹配状态。

### 5.6 数据库管理

- FR-DB-01：数据库查询：按类别、物质名称、保留时间范围、UV 波长等条件查询对照品数据。
- FR-DB-02：数据库更新：管理员按类别和数据类型（保留时间/峰面积/UV）上传文件，更新数字对照品库。
- FR-DB-03：数据库重置：将数据库恢复到上一次更新前的快照（需保留快照表或备份文件）。
- FR-DB-04：数据库导出：将当前数据库导出为 Excel 或 SQL 备份文件。
- FR-DB-05：对照品数据维护：增删改查药物类别、物质、参照物、波长配置、模型参数（期望值、方差/协方差、样本数）。

### 5.7 日志与审计

- FR-LOG-01：记录用户登录、检测、导入、导出、数据库更新等操作。
- FR-LOG-02：日志字段：操作人、操作时间、IP、操作类型、样品编号、结果摘要。
- FR-LOG-03：管理员可查看全部日志；操作员只能查看自己的日志。
- FR-LOG-04：支持按时间、用户、操作类型筛选与导出。

### 5.8 系统设置

- FR-SET-01：阈值配置：RRT 容差、UV 距离阈值、峰面积比值概率阈值。
- FR-SET-02：数据库连接配置（ host、port、用户名、密码、数据库名 ）。
- FR-SET-03：类别与波长配置：维护 8 大类的参照物和检测波长。

### 6.3 通用业务规则

- BR-01：必须先选择药物类别，系统才能加载对应的数字对照品库。
- BR-02：参照物保留时间必须正确；若输入错误，所有相对保留时间结果都会错误。
- BR-03：一个样品可包含多个峰；每个峰独立进行三种模型检测，最终再综合判定。
- BR-04：UV 模型优先级最高（准确性最高），RRT 用于初筛，峰面积比值用于辅助确认。
- BR-05：当同一峰匹配到多个物质时，按置信度/概率降序排列，由用户人工确认。
- BR-06：数据库更新后，应重新计算并保存该类别贝叶斯模型参数（期望值、方差/协方差、样本数 N）。
- BR-07：数据库重置必须可回滚，重置前自动创建快照。

## 7. 数据格式与接口约定

### 7.1 样品数据结构

```json
{
  "sample_no": "0406-1",
  "category_id": 1,
  "category_name": "安神镇静类",
  "operator_id": 2,
  "location": "实验室 A",
  "ref_substance": "盐酸氯丙嗪",
  "ref_retention_time": 12.988,
  "peaks": [
    {
      "peak_index": 1,
      "retention_time": 6.256,
      "areas": {"245": 1595667, "250": 1403031, "255": 1138550, "260": 813884},
      "uv_lambda": ["227.5"]
    }
  ]
```

## 8. 算法需求（概要）

详细算法公式、伪代码、接口规划见 `docs/algorithm_design.md`。本需求文档只做原则性约束：

- **相对保留时间**：使用一元正态分布贝叶斯模型，参数 `mu`、`sigma²`、`N`；兼容原桌面版 `|RRT - mu| <= 0.001` 的兜底匹配。
- **峰面积比值**：使用多维正态分布贝叶斯模型，参数 `mu` 向量、`Sigma` 协方差矩阵、`N`。
- **紫外吸收光谱**：使用欧氏距离阈值法，距离小于阈值判定命中。
- **综合判定**：支持“加权模式”与“严格交集模式”，最终输出 `confirmed` / `suspect` / `none`。
- **增量更新**：新增训练样本时，按贝叶斯迭代公式更新 `N`、`mu`、`Sigma`。

## 11. 验收标准

- AC-01：管理员能新增用户并分配角色；操作员只能访问授权功能。
- AC-02：能正确解析原始 Waters txt 保留时间与峰面积文件，解析成功率 ≥ 99%。
- AC-03：对已知安神类样品能正确识别出对应物质（与桌面版结果一致）。
- AC-04：多药物批量检测能一次处理 ≥ 20 个样品并生成汇总报告。
- AC-05：峰值图对比能正确展示样品峰与库中对照品峰的叠加图。
- AC-06：数据库导入/更新/重置/导出功能可用，重置后能恢复到上一次状态。
- AC-07：所有操作均记录到审计日志，且日志不可被操作员删除。

## 12. 遗留问题与待确认事项

1. **部分类别基础数据缺失**：减肥类、降压类、消肿止痛抗风湿类、止咳平喘类的参照物、检测波长、完整物质清单在提供的原始资料中不完整，需要用户提供或补充。
2. **阈值默认值**：RRT 容差、UV 距离阈值、峰面积概率阈值需要由业务专家确认默认值。当前按 `docs/algorithm_design.md` 与 `backend/sql/schema.sql` 中的建议值设置。
3. **模型训练数据**：现有 MySQL 中是否已存在完整的贝叶斯模型参数表，还是需要根据原始 xlsx/txt 重新训练，需要确认。
4. **峰面积波长配置**：不同类别使用的波长数量可能不同（如降脂类示例只有 250/254 nm），需要在 `drug_categories.wavelengths` 中灵活支持 2~4 个波长。
5. **峰值图数据来源**：是否需要从原始 HPLC 原始信号文件获取完整色谱曲线，还是仅根据峰保留时间与面积绘制示意峰，需要确认。
6. **日志导出格式**：日志导出为 txt、Excel 还是 PDF，需要确认。
7. **部署环境**：MySQL 使用本地服务器还是继续使用阿里云 RDS，需要确认网络与安全策略。
